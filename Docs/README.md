# Docs 索引

## 推荐阅读顺序

### 1. 先看系统总览

- [System-Architecture-Overview.md](./System-Architecture-Overview.md)

适合用来理解：

- 这个项目有哪些系统
- 各系统分别负责什么
- 各系统之间怎么协作
- 一局游戏从开局到结算怎么跑起来

### 2. 再看维护与扩展指南

- [Maintenance-Extension-Guide.md](./Maintenance-Extension-Guide.md)

适合用来落地扩展：

- 怎么加武器
- 怎么加技能卡
- 怎么加敌人
- 哪些东西适合资产化
- 哪些东西应该继续留在代码里

## 这三份文档分别解决什么问题

### `System-Architecture-Overview.md`

回答“这个项目现在是怎么组织起来的”。

### `Maintenance-Extension-Guide.md`

回答“我要继续做功能，具体应该改哪里”。

### `README.md`

回答“我先看哪份文档最合适”。
